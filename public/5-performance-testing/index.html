Test Failure Notifications

🔔 Test Failure Notifications using SNS and CloudWatch

In this section, you’ll learn how to set up automatic notifications for test failures using Amazon CloudWatch for monitoring and Amazon SNS for notifications.

🎯 Objectives

  • Create automatic notifications (email, SMS…) when a build fails
  • Use Amazon CloudWatch to monitor build status
  • Use Amazon SNS (Simple Notification Service) to send alerts
  • Help managers/devs know immediately when test errors occur

5.1. Create SNS Topic and Subscribe Email

Create SNS Topic

  1. Access Amazon SNS Console
  2. Create new topic:
aws sns create-topic --name test-failure-alerts

  1. Save the returned Topic ARN

Subscribe Email for Notifications

aws sns subscribe \
    --topic-arn arn:aws:sns:region:account-id:test-failure-alerts \
    --protocol email \
    --notification-endpoint your-email@example.com

You’ll receive a confirmation email. Remember to click the link in the email to confirm your subscription.

5.2. Create CloudWatch Rule

Create EventBridge Rule to Monitor CodeBuild

aws events put-rule \
    --name codebuild-failure-rule \
    --event-pattern '{
        "source": ["aws.codebuild"],
        "detail-type": ["CodeBuild Build State Change"],
        "detail": {
            "build-status": ["FAILED"]
        }
    }'

Grant CloudWatch Permission to Publish to SNS

aws sns add-permission \
    --topic-arn arn:aws:sns:region:account-id:test-failure-alerts \
    --label cloudwatch-access \
    --aws-account-id account-id \
    --action-name Publish

5.3. Connect Rule to SNS Topic

Add SNS as Rule Target

aws events put-targets \
    --rule codebuild-failure-rule \
    --targets "Id"="1","Arn"="arn:aws:sns:region:account-id:test-failure-alerts"

Customize Notification Format

{
    "build-status": "FAILED",
    "project-name": "ci-dotnet-unittest",
    "build-id": "<build-id>",
    "failure-reason": "Test failures detected",
    "logs-url": "https://console.aws.amazon.com/codebuild/home?region=region#/builds/<build-id>/view/new"
}

5.4. Verify Setup

Create a Failed Build to Test

  1. Modify a test case to fail:
[Fact]
public void Test_Should_Fail()
{
    Assert.True(false); // This will fail
}
  1. Push code and wait for CodeBuild to run

Confirm Notification Receipt

  1. Check registered email
  2. Confirm notification contains:
    • Project name
    • Build ID
    • Failure reason
    • Link to build logs

🎯 Achievements

  • ✅ Instant notifications for test failures
  • 📊 Monitor status of all builds
  • 🔍 Quick access to build logs for debugging
  • ⚡ Reduced time to detect and fix errors

You can extend the system by:

  • Adding other notification channels (Slack, MS Teams…)
  • Customizing notification format
  • Adding other types of alerts (timeout, memory issues…)
