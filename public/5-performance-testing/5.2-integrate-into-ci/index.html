Integrate into CI/CD

Integrating Performance Tests into CI/CD

Overview

In this section, we’ll integrate our performance tests into the AWS CodeBuild CI/CD pipeline. We’ll configure the build process to automatically run performance tests and evaluate the results.

Implementation Steps

1. Update BuildSpec Configuration

  1. Create performance test buildspec:
# performance-buildspec.yml
version: 0.2

phases:
  install:
    runtime-versions:
      nodejs: 18
    commands:
      - curl https://github.com/grafana/k6/releases/download/v0.45.0/k6-v0.45.0-linux-amd64.tar.gz -L | tar xvz --strip-components=1
      - mv k6 /usr/local/bin

  pre_build:
    commands:
      - echo Starting performance tests at $(date)
      - mkdir -p performance-results

  build:
    commands:
      - k6 run basic-load-test.js --out json=performance-results/load-test.json
      - k6 run stress-test.js --out json=performance-results/stress-test.json
      - k6 run collect-metrics.js --out json=performance-results/metrics.json

  post_build:
    commands:
      - echo Performance tests completed at $(date)
      - node generate-performance-report.js

reports:
  performance-reports:
    files:
      - performance-results/*.json
    file-format: JSON

artifacts:
  files:
    - performance-results/**/*
    - performance-report.html
  name: performance-test-results-$(date +%Y-%m-%d)

2. Create Performance Test Job

  1. Configure CodeBuild project:
aws codebuild create-project \
  --name performance-test-project \
  --source type=GITHUB,location=https://github.com/<username>/dotnet-test-automation.git \
  --artifacts type=S3,location=performance-test-results \
  --environment type=LINUX_CONTAINER,image=aws/codebuild/standard:7.0,computeType=BUILD_GENERAL1_LARGE \
  --service-role PerformanceTestRole
  1. Create IAM role:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:ListBucket",
        "cloudwatch:PutMetricData",
        "logs:CreateLogGroup",
        "logs:CreateLogStream",
        "logs:PutLogEvents"
      ],
      "Resource": "*"
    }
  ]
}

3. Configure Performance Thresholds

  1. Create threshold configuration:
// performance-thresholds.js
module.exports = {
  loadTest: {
    responseTime: {
      p95: 500,    // 95th percentile response time in ms
      p99: 1000    // 99th percentile response time in ms
    },
    errorRate: 0.01,     // 1% error rate threshold
    throughput: 100      // Minimum requests per second
  },
  stressTest: {
    responseTime: {
      p95: 1000,
      p99: 2000
    },
    errorRate: 0.02,
    throughput: 200
  }
};
  1. Create evaluation script:
// evaluate-performance.js
const thresholds = require('./performance-thresholds');

function evaluateResults(results) {
  const violations = [];
  
  if (results.metrics.http_req_duration.p95 > thresholds.loadTest.responseTime.p95) {
    violations.push(`P95 response time exceeded threshold: ${results.metrics.http_req_duration.p95}ms`);
  }
  
  if (results.metrics.error_rate > thresholds.loadTest.errorRate) {
    violations.push(`Error rate exceeded threshold: ${results.metrics.error_rate * 100}%`);
  }
  
  return violations;
}

module.exports = { evaluateResults };

4. Set Up Monitoring

  1. Create CloudWatch dashboard:
aws cloudwatch create-dashboard \
  --dashboard-name PerformanceTests \
  --dashboard-body file://performance-dashboard.json
  1. Configure metrics:
{
  "metrics": [
    [ "CustomMetrics", "ResponseTime", "TestType", "Load", { "stat": "p95" } ],
    [ ".", "ErrorRate", ".", ".", { "stat": "Average" } ],
    [ ".", "Throughput", ".", ".", { "stat": "Sum" } ]
  ],
  "view": "timeSeries",
  "stacked": false,
  "period": 300
}

Best Practices

  1. Resource Management

    • Use appropriate compute types
    • Monitor resource utilization
    • Clean up test data
    • Implement timeouts

  2. Test Scheduling

    • Run at consistent times
    • Avoid peak hours
    • Consider time zones
    • Set execution windows

  3. Result Handling

    • Store historical results
    • Track trends
    • Set alert thresholds
    • Document anomalies

Common Issues and Solutions

  1. Resource Constraints

    • Scale compute capacity
    • Optimize test scripts
    • Batch test execution
    • Monitor memory usage

  2. Network Issues

    • Configure timeouts
    • Handle retries
    • Log connection errors
    • Monitor bandwidth

  3. Data Management

    • Clean up test data
    • Manage storage
    • Archive results
    • Implement retention
