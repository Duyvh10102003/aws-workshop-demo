Gửi cảnh báo khi test thất bại

🔔 Gửi cảnh báo khi test thất bại bằng SNS và CloudWatch

Trong phần này, bạn sẽ học cách thiết lập hệ thống cảnh báo tự động khi có test thất bại, sử dụng Amazon CloudWatch để theo dõi và Amazon SNS để gửi thông báo.

🎯 Mục tiêu

  • Tạo thông báo tự động (email, SMS…) khi một bản build thất bại
  • Dùng Amazon CloudWatch để theo dõi trạng thái build
  • Dùng Amazon SNS (Simple Notification Service) để gửi cảnh báo
  • Giúp người quản lý/dev biết ngay khi có lỗi test xảy ra

5.1. Tạo SNS Topic và đăng ký email

Tạo SNS Topic

  1. Truy cập Amazon SNS Console
  2. Tạo topic mới:
aws sns create-topic --name test-failure-alerts

  1. Lưu lại Topic ARN được trả về

Đăng ký email nhận thông báo

aws sns subscribe \
    --topic-arn arn:aws:sns:region:account-id:test-failure-alerts \
    --protocol email \
    --notification-endpoint your-email@example.com

Bạn sẽ nhận được email xác nhận đăng ký. Nhớ click vào link trong email để xác nhận subscription.

5.2. Tạo CloudWatch Rule

Tạo EventBridge Rule để theo dõi CodeBuild

aws events put-rule \
    --name codebuild-failure-rule \
    --event-pattern '{
        "source": ["aws.codebuild"],
        "detail-type": ["CodeBuild Build State Change"],
        "detail": {
            "build-status": ["FAILED"]
        }
    }'

Cấp quyền cho CloudWatch gửi tới SNS

aws sns add-permission \
    --topic-arn arn:aws:sns:region:account-id:test-failure-alerts \
    --label cloudwatch-access \
    --aws-account-id account-id \
    --action-name Publish

5.3. Kết nối Rule với SNS Topic

Thêm SNS làm target cho Rule

aws events put-targets \
    --rule codebuild-failure-rule \
    --targets "Id"="1","Arn"="arn:aws:sns:region:account-id:test-failure-alerts"

Tùy chỉnh format thông báo

{
    "build-status": "FAILED",
    "project-name": "ci-dotnet-unittest",
    "build-id": "<build-id>",
    "failure-reason": "Test failures detected",
    "logs-url": "https://console.aws.amazon.com/codebuild/home?region=region#/builds/<build-id>/view/new"
}

5.4. Kiểm tra hoạt động

Tạo một bản build lỗi để test

  1. Sửa một test case để fail:
[Fact]
public void Test_Should_Fail()
{
    Assert.True(false); // This will fail
}
  1. Push code và đợi CodeBuild chạy

Xác nhận nhận được thông báo

  1. Kiểm tra email đã đăng ký
  2. Xác nhận nội dung thông báo chứa:
    • Tên project
    • Build ID
    • Lý do thất bại
    • Link tới build logs

🎯 Kết quả đạt được

  • ✅ Nhận thông báo tức thì khi có test fail
  • 📊 Theo dõi được trạng thái của tất cả builds
  • 🔍 Truy cập nhanh tới build logs để debug
  • ⚡ Giảm thời gian phát hiện và xử lý lỗi

Bạn có thể mở rộng hệ thống bằng cách:

  • Thêm các kênh thông báo khác (Slack, MS Teams…)
  • Tùy chỉnh format thông báo
  • Thêm các loại cảnh báo khác (timeout, memory issues…)
