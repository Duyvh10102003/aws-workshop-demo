So sánh Tốc độ

So sánh Tốc độ Thực thi

Tổng quan

Trong phần này, chúng ta sẽ phân tích và so sánh tốc độ thực thi giữa các lần chạy kiểm thử tuần tự và song song. Chúng ta sẽ đo lường cải thiện hiệu năng, xác định điểm nghẽn và tối ưu hóa thực thi kiểm thử để đạt hiệu quả tối đa.

Các bước Thực hiện

1. Tạo Công cụ Đo lường Hiệu năng

  1. Tạo script đo thời gian:
# measure-execution.ps1
param(
    [string]$TestMode = "sequential"
)

$stopwatch = [System.Diagnostics.Stopwatch]::StartNew()

if ($TestMode -eq "sequential") {
    # Chạy kiểm thử tuần tự
    Get-ChildItem -Filter "*.Tests.*" | ForEach-Object {
        dotnet test $_.FullName
    }
} else {
    # Chạy kiểm thử song song
    $jobs = Get-ChildItem -Filter "*.Tests.*" | ForEach-Object {
        Start-Job -ScriptBlock { 
            param($path)
            dotnet test $path 
        } -ArgumentList $_.FullName
    }
    Wait-Job $jobs
    Receive-Job $jobs
}

$stopwatch.Stop()
$executionTime = $stopwatch.Elapsed

return @{
    Mode = $TestMode
    Duration = $executionTime
    TimeInSeconds = $executionTime.TotalSeconds
}

2. Triển khai So sánh Hiệu năng

  1. Tạo script so sánh:
# compare-performance.py
import matplotlib.pyplot as plt
import pandas as pd

def analyze_performance(sequential_results, parallel_results):
    # Tính toán cải thiện
    speedup = sequential_results['duration'] / parallel_results['duration']
    efficiency = speedup / parallel_results['worker_count']
    
    # Tạo biểu đồ so sánh
    labels = ['Tuần tự', 'Song song']
    durations = [sequential_results['duration'], parallel_results['duration']]
    
    plt.figure(figsize=(10, 6))
    plt.bar(labels, durations)
    plt.title('So sánh Thời gian Thực thi Kiểm thử')
    plt.ylabel('Thời gian (giây)')
    
    # Thêm chú thích cải thiện
    plt.annotate(f'Tăng tốc: {speedup:.2f}x\nHiệu quả: {efficiency:.2%}',
                xy=(1, parallel_results['duration']),
                xytext=(1.2, parallel_results['duration'] * 1.2),
                arrowprops=dict(facecolor='black', shrink=0.05))
    
    plt.savefig('performance-comparison.png')
    
    return {
        'speedup': speedup,
        'efficiency': efficiency,
        'time_saved': sequential_results['duration'] - parallel_results['duration']
    }

3. Giám sát Sử dụng Tài nguyên

  1. Tạo script giám sát tài nguyên:
#!/bin/bash
# monitor-resources.sh

log_resources() {
    timestamp=$(date +"%Y-%m-%d %H:%M:%S")
    cpu_usage=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}')
    memory_usage=$(free -m | awk 'NR==2{printf "%.2f%%", $3*100/$2}')
    
    echo "$timestamp,$cpu_usage,$memory_usage" >> resource-usage.csv
}

# Giám sát mỗi 5 giây
while true; do
    log_resources
    sleep 5
done

Phân tích Hiệu năng

  1. Chỉ số Chính cần Giám sát:

    • Tổng thời gian thực thi
    • Sử dụng CPU
    • Sử dụng bộ nhớ
    • Thao tác I/O
    • Băng thông mạng

  2. Xác định Điểm nghẽn:

    • Thời gian khởi tạo kiểm thử
    • Tranh chấp tài nguyên
    • Thao tác cơ sở dữ liệu
    • Thao tác hệ thống tệp

  3. Cơ hội Tối ưu hóa:

    • Chiến lược nhóm kiểm thử
    • Phân bổ tài nguyên
    • Sử dụng bộ nhớ đệm
    • Hiệu quả dọn dẹp dữ liệu

Các Thực hành Tốt nhất

  1. Đo lường Hiệu năng

    • Sử dụng dữ liệu kiểm thử nhất quán
    • Chạy nhiều lần lặp
    • Tính đến thời gian khởi động
    • Xem xét yếu tố môi trường

  2. Quản lý Tài nguyên

    • Giám sát tài nguyên hệ thống
    • Tối ưu hóa phân bổ tài nguyên
    • Xử lý dọn dẹp tài nguyên
    • Ngăn chặn rò rỉ tài nguyên

  3. Báo cáo

    • Bao gồm tất cả chỉ số liên quan
    • Trực quan hóa so sánh
    • Ghi chép bất thường
    • Theo dõi xu hướng theo thời gian

Vấn đề Thường gặp và Giải pháp

  1. Điểm nghẽn Tài nguyên

    • Triển khai pool tài nguyên
    • Tối ưu hóa dữ liệu kiểm thử
    • Sử dụng kích thước compute phù hợp
    • Giám sát giới hạn tài nguyên

  2. Phụ thuộc Kiểm thử

    • Giảm thiểu tài nguyên chia sẻ
    • Triển khai cô lập phù hợp
    • Xử lý dọn dẹp hiệu quả
    • Quản lý thứ tự kiểm thử

  3. Suy giảm Hiệu năng

    • Giám sát thường xuyên
    • Phân tích xu hướng
    • Tối ưu hóa chủ động
    • Mở rộng tài nguyên
