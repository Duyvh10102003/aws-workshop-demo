Kích hoạt CodeQL

Kích hoạt Phân tích CodeQL

Tổng quan

Trong phần này, chúng ta sẽ kích hoạt và cấu hình phân tích bảo mật CodeQL trong GitHub repository và tích hợp nó với AWS CodeBuild. CodeQL sẽ giúp chúng ta xác định các lỗ hổng bảo mật tiềm ẩn trong mã nguồn.

Các bước Thực hiện

1. Kích hoạt CodeQL trong GitHub

  1. Cấu hình workflow GitHub Actions:
# .github/workflows/codeql-analysis.yml
name: "Phân tích CodeQL"

on:
  push:
    branches: [ main, development ]
  pull_request:
    branches: [ main ]
  schedule:
    - cron: '0 0 * * 0'

jobs:
  analyze:
    name: Phân tích
    runs-on: ubuntu-latest
    permissions:
      security-events: write
      actions: read

    strategy:
      fail-fast: false
      matrix:
        language: [ 'csharp' ]

    steps:
    - name: Checkout repository
      uses: actions/checkout@v3

    - name: Khởi tạo CodeQL
      uses: github/codeql-action/init@v2
      with:
        languages: ${{ matrix.language }}

    - name: Tự động build
      uses: github/codeql-action/autobuild@v2

    - name: Thực hiện Phân tích CodeQL
      uses: github/codeql-action/analyze@v2

2. Cấu hình CodeQL trong AWS CodeBuild

  1. Cập nhật buildspec.yml:
version: 0.2

phases:
  install:
    runtime-versions:
      dotnet: 8.0
    commands:
      - wget https://github.com/github/codeql-action/releases/latest/download/codeql-bundle-linux64.tar.gz
      - tar xzf codeql-bundle-linux64.tar.gz
      
  pre_build:
    commands:
      - export PATH=$PATH:$CODEBUILD_SRC_DIR/codeql
      - codeql database create db --language=csharp
      
  build:
    commands:
      - dotnet build
      - codeql database analyze db --format=sarif-latest --output=results.sarif
      
  post_build:
    commands:
      - aws s3 cp results.sarif s3://${BUCKET_NAME}/security-scans/$(date +%Y-%m-%d)/

reports:
  security-reports:
    files:
      - results.sarif
    file-format: SARIF

3. Cấu hình Bộ Truy vấn CodeQL

  1. Tạo bộ truy vấn tùy chỉnh:
# .github/codeql/custom-queries.qls
- import: codeql-suites/security-extended.qls
- queries: .
- include:
    tags contain: security
    precision > high
- exclude:
    tags contain: experimental
  1. Thêm truy vấn tùy chỉnh:
/**
 * @name Cấu hình Không An toàn
 * @description Xác định các cài đặt cấu hình có thể không an toàn
 * @kind problem
 * @problem.severity warning
 * @security-severity 5.0
 * @tags security
 */
import csharp

from Configuration config
where config.isInsecure()
select config, "Phát hiện cấu hình có thể không an toàn"

4. Thiết lập Cảnh báo Bảo mật

  1. Cấu hình cài đặt cảnh báo trong GitHub:

    • Vào Cài đặt repository
    • Điều hướng đến Bảo mật & phân tích
    • Kích hoạt Cảnh báo quét mã
    • Cấu hình cài đặt thông báo

  2. Tạo workflow xử lý cảnh báo:

name: "Xử lý Cảnh báo Bảo mật"

on:
  workflow_dispatch:
  security_events:
    types: [created]

jobs:
  triage:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Xử lý Cảnh báo Bảo mật
        uses: actions/github-script@v6
        with:
          script: |
            const alert = context.payload.alert
            if (alert.severity === 'high') {
              await github.issues.create({
                owner: context.repo.owner,
                repo: context.repo.repo,
                title: `Cảnh báo Bảo mật: ${alert.rule.name}`,
                body: `Phát hiện vấn đề bảo mật nghiêm trọng:\n${alert.rule.description}`
              })
            }

Các Thực hành Tốt nhất

  1. Cấu hình Truy vấn

    • Bắt đầu với truy vấn mặc định
    • Thêm truy vấn tùy chỉnh dần dần
    • Tập trung vào vấn đề quan trọng
    • Cập nhật truy vấn thường xuyên

  2. Quản lý Cảnh báo

    • Đặt ngưỡng mức độ nghiêm trọng
    • Cấu hình thông báo
    • Theo dõi thời gian giải quyết
    • Ghi chép cảnh báo sai

  3. Thiết lập Tích hợp

    • Lịch quét thường xuyên
    • Quy tắc bảo vệ nhánh
    • Phân bổ tài nguyên
    • Lưu trữ kết quả

Vấn đề Thường gặp và Giải pháp

  1. Vấn đề Hiệu năng

    • Tối ưu phạm vi quét
    • Lập lịch trong giờ thấp điểm
    • Sử dụng phân tích gia tăng
    • Giám sát sử dụng tài nguyên

  2. Cảnh báo Sai

    • Ghi chép ngoại lệ
    • Cập nhật bộ truy vấn
    • Sử dụng file cơ sở
    • Quy trình xem xét thường xuyên

  3. Vấn đề Tích hợp

    • Kiểm tra quyền
    • Xác minh cấu hình
    • Giám sát log build
    • Kiểm thử cục bộ trước

Bước tiếp theo

Sau khi kích hoạt CodeQL, tiếp tục với Xem xét Cảnh báo để học cách phân tích và phân loại cảnh báo bảo mật.
