Cấu hình Cài đặt

Cấu hình Cài đặt Bảo mật

Tổng quan

Trong phần này, chúng ta sẽ học cách cấu hình và quản lý cài đặt quét bảo mật trong CodeQL và AWS CodeBuild. Chúng ta sẽ tìm hiểu cách tùy chỉnh cấu hình quét, quản lý cảnh báo sai và tối ưu hóa hiệu năng quét.

Các bước Thực hiện

1. Cấu hình Cài đặt CodeQL

  1. Tạo file cấu hình CodeQL:
# .github/codeql/codeql-config.yml
name: "Cấu hình CodeQL Tùy chỉnh"

queries:
  - uses: security-extended
  - uses: security-and-quality

paths:
  - src
  - test
paths-ignore:
  - node_modules
  - '**/*.test.js'

query-filters:
  - exclude:
      tags contain: experimental

security-severity-overrides:
  - pattern: js/sql-injection
    security-severity: 9.0
  - pattern: js/xss
    security-severity: 8.5

2. Thiết lập Lịch Quét

  1. Tạo cấu hình lịch quét:
# .github/workflows/security-scan-schedule.yml
name: Quét Bảo mật Định kỳ

on:
  schedule:
    - cron: '0 2 * * *'  # Chạy lúc 2 giờ sáng hàng ngày
  workflow_dispatch:      # Cho phép kích hoạt thủ công

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Khởi tạo CodeQL
        uses: github/codeql-action/init@v2
        with:
          config-file: .github/codeql/codeql-config.yml
          
      - name: Chạy Quét Bảo mật
        uses: github/codeql-action/analyze@v2
        with:
          category: "/language:csharp"

3. Cấu hình Cài đặt Cảnh báo

  1. Tạo cấu hình cảnh báo:
// security-alerts-config.js
const alertConfig = {
  severityLevels: {
    critical: {
      threshold: 9.0,
      notification: 'ngay-lap-tuc',
      autoAssign: true,
      assignee: 'nhom-bao-mat'
    },
    high: {
      threshold: 7.0,
      notification: 'hang-ngay',
      autoAssign: true,
      assignee: 'truong-nhom-phat-trien'
    },
    medium: {
      threshold: 4.0,
      notification: 'hang-tuan',
      autoAssign: false
    },
    low: {
      threshold: 0.0,
      notification: 'hang-tuan',
      autoAssign: false
    }
  },
  
  suppressions: {
    patterns: [
      {
        rule: 'js/sql-injection',
        paths: ['**/test/**']
      },
      {
        rule: 'js/xss',
        justification: 'Chỉ dùng cho công cụ quản trị nội bộ'
      }
    ]
  }
};

4. Tối ưu hóa Hiệu năng

  1. Tạo cài đặt tối ưu hóa:
{
  "scan": {
    "threads": 4,
    "ram": "8GB",
    "timeout": 3600,
    "incremental": true,
    "cache": {
      "enabled": true,
      "location": ".codeql/cache",
      "retention": "7d"
    }
  },
  
  "analysis": {
    "batchSize": 1000,
    "parallel": true,
    "maxFiles": 10000,
    "exclude": [
      "**/*.generated.cs",
      "**/obj/**",
      "**/bin/**"
    ]
  }
}

Các Thực hành Tốt nhất

  1. Cấu hình Quét

    • Tập trung vào truy vấn liên quan
    • Loại trừ file kiểm thử
    • Cấu hình đường dẫn phù hợp
    • Đặt ngưỡng mức độ nghiêm trọng

  2. Quản lý Hiệu năng

    • Bật quét gia tăng
    • Sử dụng bộ nhớ đệm
    • Cấu hình kích thước lô
    • Tối ưu số lượng luồng

  3. Quản lý Cảnh báo

    • Cấu hình mức độ nghiêm trọng
    • Thiết lập thông báo
    • Quản lý loại trừ
    • Ghi chép ngoại lệ

Vấn đề Thường gặp và Giải pháp

  1. Hiệu năng Quét

    • Tối ưu phạm vi quét
    • Cấu hình bộ nhớ đệm
    • Sử dụng phân tích gia tăng
    • Lập lịch trong giờ thấp điểm

  2. Cảnh báo Sai

    • Cấu hình loại trừ
    • Cập nhật bộ lọc truy vấn
    • Ghi chép ngoại lệ
    • Xem xét thường xuyên

  3. Sử dụng Tài nguyên

    • Cấu hình giới hạn luồng
    • Đặt ràng buộc bộ nhớ
    • Tối ưu kích thước lô
    • Giám sát sử dụng

Bước tiếp theo

Sau khi cấu hình cài đặt bảo mật, tiếp tục với Giám sát Chi phí để học cách giám sát và tối ưu hóa chi phí của cơ sở hạ tầng kiểm thử bảo mật.
