CloudWatch Logs

Thiết lập CloudWatch Logs

Tổng quan

Trong phần này, chúng ta sẽ cấu hình CloudWatch Logs để giám sát cơ sở hạ tầng kiểm thử và tạo các chỉ số cho phân tích chi phí. Chúng ta sẽ thiết lập nhóm log, tạo bộ lọc chỉ số và cấu hình cảnh báo cho các sự kiện liên quan đến chi phí.

Các bước Thực hiện

1. Cấu hình Nhóm Log

  1. Tạo nhóm log sử dụng AWS CLI:
# Tạo nhóm log cho dự án CodeBuild
aws logs create-log-group \
  --log-group-name /aws/codebuild/test-automation \
  --tags Project=TestAutomation,Environment=Production

# Tạo nhóm log cho thực thi kiểm thử
aws logs create-log-group \
  --log-group-name /aws/test-execution \
  --tags Project=TestAutomation,Environment=Production
  1. Đặt chính sách lưu giữ:
# Đặt thời gian lưu giữ 30 ngày cho log CodeBuild
aws logs put-retention-policy \
  --log-group-name /aws/codebuild/test-automation \
  --retention-in-days 30

# Đặt thời gian lưu giữ 14 ngày cho log thực thi kiểm thử
aws logs put-retention-policy \
  --log-group-name /aws/test-execution \
  --retention-in-days 14

2. Tạo Bộ lọc Chỉ số

  1. Tạo chỉ số sử dụng tài nguyên:
# Tạo bộ lọc chỉ số sử dụng CPU
aws logs put-metric-filter \
  --log-group-name /aws/codebuild/test-automation \
  --filter-name cpu-usage \
  --filter-pattern "[timestamp, requestId, cpu=*]" \
  --metric-transformations \
      metricName=CPUUtilization,metricNamespace=TestAutomation,metricValue=$cpu

# Tạo bộ lọc chỉ số sử dụng bộ nhớ
aws logs put-metric-filter \
  --log-group-name /aws/codebuild/test-automation \
  --filter-name memory-usage \
  --filter-pattern "[timestamp, requestId, memory=*]" \
  --metric-transformations \
      metricName=MemoryUtilization,metricNamespace=TestAutomation,metricValue=$memory

3. Thiết lập Log Insights

  1. Tạo truy vấn CloudWatch Insights:
# Truy vấn mẫu sử dụng tài nguyên
fields @timestamp, @message
| filter @message like /Resource usage:/
| parse @message "CPU: * Memory: *" as cpu, memory
| stats avg(cpu) as avgCPU, max(cpu) as maxCPU,
        avg(memory) as avgMemory, max(memory) as maxMemory
        by bin(1h)
| sort @timestamp desc
  1. Tạo truy vấn phân tích chi phí:
# Truy vấn thời gian build và chi phí
fields @timestamp, @message
| filter @message like /Build completed/
| parse @message "Duration: * Cost: $*" as duration, cost
| stats sum(cost) as totalCost,
        avg(duration) as avgDuration
        by bin(24h)
| sort @timestamp desc

4. Cấu hình Cảnh báo

  1. Tạo cảnh báo CloudWatch:
# Tạo cảnh báo chi phí cao
aws cloudwatch put-metric-alarm \
  --alarm-name ChiPhiKiemThuCao \
  --alarm-description "Cảnh báo khi chi phí kiểm thử hàng ngày vượt ngưỡng" \
  --metric-name DailyTestingCost \
  --namespace TestAutomation \
  --statistic Sum \
  --period 86400 \
  --threshold 100 \
  --comparison-operator GreaterThanThreshold \
  --evaluation-periods 1 \
  --alarm-actions arn:aws:sns:region:account-id:cost-alerts

# Tạo cảnh báo sử dụng tài nguyên cao
aws cloudwatch put-metric-alarm \
  --alarm-name SuDungTaiNguyenCao \
  --alarm-description "Cảnh báo khi mức sử dụng tài nguyên cao" \
  --metric-name CPUUtilization \
  --namespace TestAutomation \
  --statistic Average \
  --period 300 \
  --threshold 80 \
  --comparison-operator GreaterThanThreshold \
  --evaluation-periods 3 \
  --alarm-actions arn:aws:sns:region:account-id:resource-alerts

Các Thực hành Tốt nhất

  1. Quản lý Log

    • Sử dụng log có cấu trúc
    • Đặt thời gian lưu giữ phù hợp
    • Gắn thẻ nhóm log
    • Giám sát khối lượng log

  2. Cấu hình Chỉ số

    • Tạo chỉ số liên quan
    • Sử dụng đơn vị phù hợp
    • Đặt ngưỡng có ý nghĩa
    • Giám sát chất lượng chỉ số

  3. Thiết lập Cảnh báo

    • Cấu hình cảnh báo có thể thực hiện
    • Đặt ngưỡng phù hợp
    • Xác định quyền sở hữu rõ ràng
    • Tài liệu hóa quy trình phản hồi

Vấn đề Thường gặp và Giải pháp

  1. Khối lượng Log

    • Cấu hình lưu giữ log
    • Sử dụng lấy mẫu khi thích hợp
    • Lọc log không cần thiết
    • Giám sát chi phí lưu trữ

  2. Độ chính xác Chỉ số

    • Xác thực bộ lọc chỉ số
    • Kiểm tra định dạng log
    • Giám sát dữ liệu thiếu
    • Hiệu chuẩn thường xuyên

  3. Quản lý Cảnh báo

    • Giảm mệt mỏi cảnh báo
    • Đặt ngưỡng phù hợp
    • Cấu hình kênh thông báo
    • Tài liệu hóa quy trình leo thang

Bước tiếp theo

Sau khi thiết lập CloudWatch logs, tiếp tục với Phân tích Chi phí để học cách phân tích và diễn giải dữ liệu chi phí đã thu thập.
