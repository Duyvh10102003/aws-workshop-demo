Compare Speed

Comparing Execution Speed

Overview

In this section, we’ll analyze and compare the execution speed between sequential and parallel test runs. We’ll measure performance improvements, identify bottlenecks, and optimize test execution for maximum efficiency.

Implementation Steps

1. Create Performance Measurement Tools

  1. Create timing script:
# measure-execution.ps1
param(
    [string]$TestMode = "sequential"
)

$stopwatch = [System.Diagnostics.Stopwatch]::StartNew()

if ($TestMode -eq "sequential") {
    # Run tests sequentially
    Get-ChildItem -Filter "*.Tests.*" | ForEach-Object {
        dotnet test $_.FullName
    }
} else {
    # Run tests in parallel
    $jobs = Get-ChildItem -Filter "*.Tests.*" | ForEach-Object {
        Start-Job -ScriptBlock { 
            param($path)
            dotnet test $path 
        } -ArgumentList $_.FullName
    }
    Wait-Job $jobs
    Receive-Job $jobs
}

$stopwatch.Stop()
$executionTime = $stopwatch.Elapsed

return @{
    Mode = $TestMode
    Duration = $executionTime
    TimeInSeconds = $executionTime.TotalSeconds
}

2. Implement Performance Comparison

  1. Create comparison script:
# compare-performance.py
import matplotlib.pyplot as plt
import pandas as pd

def analyze_performance(sequential_results, parallel_results):
    # Calculate improvements
    speedup = sequential_results['duration'] / parallel_results['duration']
    efficiency = speedup / parallel_results['worker_count']
    
    # Create comparison chart
    labels = ['Sequential', 'Parallel']
    durations = [sequential_results['duration'], parallel_results['duration']]
    
    plt.figure(figsize=(10, 6))
    plt.bar(labels, durations)
    plt.title('Test Execution Time Comparison')
    plt.ylabel('Duration (seconds)')
    
    # Add improvement annotations
    plt.annotate(f'Speedup: {speedup:.2f}x\nEfficiency: {efficiency:.2%}',
                xy=(1, parallel_results['duration']),
                xytext=(1.2, parallel_results['duration'] * 1.2),
                arrowprops=dict(facecolor='black', shrink=0.05))
    
    plt.savefig('performance-comparison.png')
    
    return {
        'speedup': speedup,
        'efficiency': efficiency,
        'time_saved': sequential_results['duration'] - parallel_results['duration']
    }

3. Monitor Resource Usage

  1. Create resource monitoring script:
#!/bin/bash
# monitor-resources.sh

log_resources() {
    timestamp=$(date +"%Y-%m-%d %H:%M:%S")
    cpu_usage=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}')
    memory_usage=$(free -m | awk 'NR==2{printf "%.2f%%", $3*100/$2}')
    
    echo "$timestamp,$cpu_usage,$memory_usage" >> resource-usage.csv
}

# Monitor every 5 seconds
while true; do
    log_resources
    sleep 5
done

    • Total execution time

    • Network bandwidth

  2. Bottleneck Identification:

    • Test initialization time
    • Resource contention

  3. Optimization Opportunities:

    • Test grouping strategies
    • Resource allocation
    • Cache utilization
    • Data cleanup efficiency

  1. Performance Measurement

    • Use consistent test data
    • Run multiple iterations

    • Consider environmental factors

  2. Resource Management

    • Handle resource cleanup
    • Prevent resource leaks

  3. Reporting

    • Include all relevant metrics
    • Visualize comparisons
    • Document anomalies

Common Issues and Solutions

  1. Resource Bottlenecks

    • Implement resource pooling
    • Optimize test data

    • Monitor resource limits

  2. Test Dependencies

    • Minimize shared resources
    • Implement proper isolation
    • Handle cleanup efficiently
    • Manage test order

  3. Performance Degradation

    • Regular monitoring
    • Trend analysis
    • Proactive optimization
    • Resource scaling
